Lesson 3 Highlights

Andrew Jones

Highlight #1 Version Control

Version Control is the management of changes to documents, computer programs, large web sites, and other collections of information. Essentially it is One main repository for all of the project files. Version Control is useful for graphic designers that are trying to save every stage of an image design they credited. A version control system also keeps all of the changes on one system and allows the owner to revert back to past changes, or to see who has made changes to something.

Highlight #2 GIT

Git is a version control system that tracks changes in computer files. Git is mainly used for software development, however it can track changes to any computer file. Git was created in 2005 by Linus Torvalds for the development of the Linux kernel. It is free and open sourced which makes it easy to learn, and has lightning fast performance.

Highlight #3 Remote Repository

Remote repository is a repository that resides on a remote server that is shared among multiple team members. This always multiple computers to access the repository in order to share files. Must be connected to a local repository in order to work.

Highlight #4 Local Repository

A local repository is one that can only be accessed by you. This means that the files and changes are saved directly to your computer and cannot be accessed by another computer, like a remote repository.

Highlight #5 Distributed version control system

In a distributed version control system every user has a complete copy of the repository data stored locally on their machine. Also everything is local and can be easily accessed. Finally, Git will prevent an unintentional overwrite of a file on the shared repository.
